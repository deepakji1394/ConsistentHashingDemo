How Consistent Hashing Works

  • Ring: Hash space 0–3600 arranged as a circle. Every server and key maps to a point on it.
  • Server placement: Each virtual node is placed at FNV-1a("name#i") mod 3600. Different indices spread nodes across the ring independently.
  • Key routing: A key at position P routes to the first server node found scanning clockwise from P. If none exists before the end, it wraps back to 0.
  • Add server: New nodes appear on the ring. Keys that are clockwise-between the previous node and the new node now route to the new server instead → only a subset migrates.
  • Remove server: Its nodes vanish. Each orphaned key rescans clockwise and lands on the next remaining node → keys spread to neighbours, not everyone.
  • Virtual nodes: More positions per server ≈ smoother load distribution. Without them, unequal spacing causes hotspots.
